## Proposed definition (relative to impact; low-friction)

### “Materially change” (operational meaning)
A decision is substantive if it is reasonably likely to change any of the following (even if the code still “works”):
- **Results**: numeric outputs, rankings, classifications, maps, estimates, uncertainty.
- **Conclusions**: interpretation, recommendations, causal claims, generalization.
- **Scope**: who/what/where/when the analysis applies to.
- **Risk**: privacy/security/legal/policy exposure; reputational or operational risk.
- **Reproducibility/maintainability**: hidden dependencies, non-determinism, hard-to-audit logic.

## Impact-graded “decision tiers” (to manage friction)
To avoid interrupting on everything, define tiers the AI platform should use:

- **Tier 1 (must interrupt + require user choice)**  
  Anything that affects results/conclusions/actions, changes risk posture, or adds a new assumption due to missing info.

- **Tier 2 (should flag, but can proceed with a clearly labeled default unless user objects)**  
  Choices that affect maintainability, performance, or interpretability *without* changing meaning/results (e.g., refactor structure, style, minor efficiency changes). The AI should still disclose the choice and its tradeoffs.

- **Tier 3 (non-substantive; no flag needed)**  
  Purely cosmetic or mechanical changes with no effect on meaning/results (formatting, comments, spelling), and changes fully determined by existing repo conventions.

This is the key “A with less friction” mechanism: **interrupt only at Tier 1**.
